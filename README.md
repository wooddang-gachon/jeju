# 🍃 제주의 숨 (Jeju Breath)

> **제주 바이오 AX 해커톤 참가작 (헬스케어 부문)**  
> **제주를 방문하는 여행객 러너(Run-Traveler)를 위한 AI 기반 맞춤형 웰니스 런트립(Runtrip) & 바이오 리커버리 웹 서비스**

---

## 1. 프로젝트 소개 (About the Project)

**"가장 제주다운 길을 달리며, 내 몸에 가장 건강한 제주를 마시다."**

**제주의 숨 (Jeju Breath)**은 제주의 실시간 환경 공공데이터(바람, 미세먼지, 자외선)와 생성형 AI 기술을 결합하여, 여행객들이 날씨 제약 없이 쾌적하게 런닝을 즐기고 제주의 숨겨진 비경을 탐험하도록 돕는 헬스케어 플랫폼입니다. 

러닝 후에는 사용자의 피로 상태와 생체 데이터를 분석하여 식의약처 데이터 기반의 제주산 천연 원료 기능성 식품(리커버리 푸드)을 추천하고, 로컬 소상공인과의 협력(짐 보관, 샤워 등)을 원스톱 QR 패스로 연결하는 상생 웰니스 생태계를 지향합니다.

---

## 2. 핵심 기능 (Key Features)

### 🗺️ AI 실시간 코스 빌더 (AI-Powered Wellness Route Builder)
* **숙소 위치 기반 다이내믹 라우팅**: 오늘 머무는 숙소를 기준으로 사용자가 원하는 목표 거리(3km, 5km 등)에 맞춘 왕복/편도 코스 자동 설계.
* **실시간 웰니스 필터링**: 실시간 자외선 지수, 바람 세기/방향, 대기질 데이터를 종합 분석하여 맞바람을 피하고 자외선 노출을 최소화하며 피톤치드 산소 농도가 높은 최적의 숲길/오솔길/밭담길로 우회 설계.

### 🌫️ 게이미피케이션 탐험 지도 (Fog of War Exploration)
* **제주 영토 개척 게임성**: 아직 가보지 않은 제주의 코스는 지도 위에 안개(Fog of War)로 덮여 있습니다. GPS를 켜고 실제 코스를 달리면 캐릭터 반경의 안개가 실시간으로 걷히며 밝은 지도로 바뀝니다.
* **제주 탐험률 & 웰니스 배지**: 나의 제주 영토 탐험률(%)이 누적으로 기록되며, 테마별 코스 완주 시 제주 고유의 웰니스 배지(예: 밭담 가디언, 피톤치드 헌터 등)를 획득합니다.

### 🥤 바이오 리커버리 식단 매칭 & 러너 패스 (Runner's Pass)
* **생체 데이터 기반 즉석 리커버리 처방**: 러닝 후 소모한 에너지와 심박수를 분석하여 식의약 데이터포털 성분 기반의 최적 제주 로컬 웰빙 메뉴(예: 전해질 보충용 제주 청귤 에이드, 피로 해소에 좋은 제주 메밀 요리 등)를 매칭합니다.
* **원스톱 오프라인 러너 패스**: 코스의 종착지 근처 제휴 상점에서 짐 보관, 샤워실 이용, 리커버리 메뉴 할인을 스마트폰 QR 코드(러너 패스) 하나로 간편하게 이용합니다.

## 3. 활용 공공데이터 (Public Data Integration)

* **제주데이터허브**:
  * 제주 올레길, 숲길, 오름 도보 경로(GPX) 데이터
  * 오름 및 숲길 주변 산소 농도, 피톤치드 방출량 데이터
  * 제주 로컬 음식점, 카페, 문화 공간 위치(위경도) 데이터
* **기상청 공공데이터**:
  * 제주 관측소(AWS) 실시간 기온, 강수량, 자외선(UV) 지수, 풍향/풍속 데이터
* **에어코리아**:
  * 실시간 미세먼지(PM10) 및 초미세먼지(PM2.5) 대기질 측정 데이터
* **식품의약품안전처 (식의약데이터포털)**:
  * 건강기능식품 개별인정형 원료 성분 및 기능성 데이터 (제주 특산 원료 매칭용)
  * 전국 식품 영양성분 정보 DB (리커버리 식단 설계용)

## 4. 시스템 아키텍처 및 기술 스택 (Architecture & Tech Stack)

### Architecture Flow
1. **Data Collection**: 기상청/에어코리아/제주데이터허브 API를 통한 실시간 환경 정보 동적 수집.
2. **AI Recommendation (Gemini API)**: 환경 변수 + 사용자 숙소 위치 + 개인 헬스 목표 + 완주 심박수를 LLM 프롬프트에 주입하여 다이내믹 코스 및 개인 맞춤형 리커버리 푸드 추천 코멘트 생성.
3. **Web Platform rendering**: React 기반 반응형 웹 환경에서 카카오/네이버 지도 API를 활용해 GPX 트랙을 렌더링하고, 안개 효과 레이어를 캔버스 위에 구현.
4. **Local Integration (Firebase)**: Firebase Auth를 통한 소셜 로그인 및 Firestore를 사용한 사용자 탐험 지표, 배지 획득 이력 저장.

### Tech Stack
* **Frontend**: React (Vite)
* **Styling**: Vanilla CSS (Premium Wellness Glassmorphism UI)
* **Database & Authentication**: Firebase (Firestore / Auth)
* **AI Engine**: Gemini API (Generative AI Consultation)
* **Maps API**: Kakao Map / Naver Map Web SDK
