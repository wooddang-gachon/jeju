# 제주 로컬 런트립 코스 추천 에이전트 기획서

본 문서는 제주(Jeju) 프로젝트에서 제공하는 단발성(Single-turn) 호출 기반의 '제주 로컬 런트립 코스 추천 에이전트'의 아키텍처 및 상세 설계서입니다. 이 에이전트는 기존의 규칙 기반 추천 필터링 함수를 AI 기반 에이전틱 워크플로우로 대체하여, 사용자에게 가장 감성적이고 정교한 로컬 러닝 경험을 선사합니다.

---

## 1. 프로젝트 개요 (Overview)
* **서비스명**: 제주 로컬 런트립 가이드 (RunJeju Agent)
* **목적**: 사용자의 출발지와 취향 데이터를 기반으로 제주 고유의 매력을 느낄 수 있는 숨겨진 러닝 코스, 로컬 맛집, 그리고 샤워/용천수 노천탕 스팟을 하나로 엮은 완결된 '런트립 코스'를 추천합니다.
* **차별화 포인트 (로컬 밀착)**:
  - 뻔한 관광 코스(탑동광장, 이호테우 등)가 아닌, 돌담밭길, 한적한 포구, 밭담 농로 등 숨겨진 로컬 길 발굴.
  - 러닝 종점 부근의 도민 추천 노포(고기국수, 몸국, 해녀의 집) 연계.
  - 제주 특유의 용천수 노천탕을 연계하여 달린 후 씻을 수 있는 실용성 제공.

---

## 2. 에이전트 아키텍처 (Agent Architecture)
본 에이전트는 **단발성 호출(Single-turn Request/Response) 함수형 에이전트**로 동작합니다. 사용자의 입력을 받으면 에이전트 내부의 여러 도구(Tools)를 순차적 혹은 병렬적으로 호출한 뒤, 완성된 추천 결과 객체를 한 번에 반환합니다.

### 워크플로우 파이프라인 (기상 위험 조건 제어 포함)
1. **[입력 수신]** 사용자 프로필 및 취향 파라미터 접수.
2. **[기상 위험 감지 (if-else 조건 분기)]** 
   - `if` (당일 기상 예보가 폭우/폭설/폭염 경보 등 러닝 위험군에 속하는가?) ➔ **'대체 일자 추천 워크플로우'**로 분기.
   - `else` ➔ 정상적인 **'야외 코스 추천 워크플로우'** 진행.
3. **[코스 후보 탐색]** (정상 시) 출발지 반경 내 보행 안전성이 검증된 로컬 경로 데이터베이스 및 위성 정보 탐색.
4. **[F&B 및 편의시설 매핑]** (정상 시) 코스 피니시 지점 기준 인근 도민 노포 맛집 및 목욕/용천수 스팟 매핑.
5. **[날씨 기반 최적화]** (정상 시) 실시간 제주 날씨 및 바람 정보를 조회하여 맞바람 구간을 최소화하도록 코스 방향 조정.
6. **[우회 정보 취합]** (위험 시) 기상 예보 API를 통해 향후 3~4일 중 가장 러닝하기 쾌적한 대체 날짜와 기온 정보 탐색.
7. **[리포트 생성]** 최종 정상 추천 코스 리포트 혹은 날씨 우회 권장 리포트 생성 및 반환.

## 3. 에이전트 상세 설계 (Agent Specification)

### 3.1 입력 스키마 (Input JSON)
```json
{
  "start_location": "구좌읍 평대리",
  "running_distance_km": 5.0,
  "runner_level": "Intermediate",
  "preferred_theme": "sea",
  "after_run_preference": "soup_and_shower"
}
```

  "course_path_summary": "평대리 포구 -> 평대리 밭담 농로 -> 세화 해안도로 -> 평대리 포구 복귀",

  "local_experience": {
    "restaurant_name": "평대 웅이네 해녀의 집",
    "menu_recommendation": "뜨끈한 전복죽과 소라 물회",

## 4. 도구 설계 (Tool Integration)

에이전트는 내부적으로 다음 핵심 도구(Custom Tools)를 활용하여 데이터를 취합하고 판단을 내립니다.

1. **`SearchLocalRunningPath`**
   - 역할: 출발지 주변의 올레길, 밭담 농로, 해안도로 노면 상태 및 안전성 조회.
2. **`FindJejuAuthenticFood`**
   - 역할: 인근 도민 평점이 높고 광고성 리뷰가 없는 로컬 노포 식당 및 해녀의 집 필터링.
3. **`MapSpringWaterAndShowers`**
   - 역할: 코스 종점 주변의 자연 용천수탕, 해수욕장 공공 샤워실, 로컬 대중목욕탕 정보 매핑.
4. **`GetJejuWindAndWeather`**
   - 역할: 실시간 풍향/풍속, 기온 및 기상 특보(호우, 폭설, 폭염 등) 정보를 파악하여 안전 등급을 평가하고 러닝 방향을 최적화함.

## 5. UI/UX 연계 방안
* **조건부 렌더링**: 프론트엔드는 에이전트가 반환한 `status` 값에 따라 다른 UI 컴포넌트를 렌더링합니다.
  - `status == "success"` ➔ 지도가 포함된 러닝 코스 요약 카드 노출.
  - `status == "warning"` ➔ 빨간색 기상 경고창과 함께 대체 날짜로 스케줄을 변경할 수 있는 '예약 일정 변경' 팝업 노출.

---

## 6. 평가 지표 (Evaluation Metrics)
* **응답 속도 (Latency)**: 사용자가 입력을 넣고 최종 코스를 받기까지의 소요 시간 (목표: 5초 이내)
* **기상 조건 탐지율**: 기상 특보 발령 시 100% 확률로 경고 포맷을 생성해 내는지 확인하는 단위 테스트 수행.
* **코스 정합성**: 추천된 러닝 경로가 실제로 차량 위험이 없고 쾌적하게 뛸 수 있는 길인지 여부 (정기적인 GPX 로그 검증)
* **로컬성 만족도**: 추천된 맛집 및 편의시설에 대한 사용자 긍정 피드백 비율
